## **Проект: Автоматизированное тестирование API (StellarBurgers)**  
 Этот проект содержит набор автотестов для API сервиса StellarBurgers.  
 Тесты написаны на языке Python с использованием фреймворков pytest и Allure для отчетов.

**🛠️ Технологии и инструменты**  
Язык программирования: Python 3.14  
Фреймворк тестирования: pytest  
Управление окружением: venv (виртуальное окружение)  
Генерация отчетов: allure-pytest  
HTTP-клиент: requests  

## 📁 Структура проекта

Проект построен по принципу разделения ответственности: клиенты API, тестовые сценарии, данные и вспомогательные утилиты.

```plain
API-tests/
├── README.md                 # Описание проекта и инструкция по запуску
├── pytest.ini                # Конфигурация pytest
├── requirements.txt          # Зависимости проекта
├── .gitignore                # Игнорируемые файлы для Git
├── allure-reports/           # Готовые HTML‑отчёты Allure (после генерации)
├── helpers/                  # Вспомогательные функции и утилиты
│   ├── generate_data.py      # Генераторы данных (адрес, пароль, имя)
│   └── user_generate_data.py # Генерация payload для нового пользователя
├── data/                     # Константы и шаблоны данных
│   ├── constants.py          # URL сервиса, сообщения об ошибках
│   └── payload_templates.py  # Шаблоны payload для регистрации пользователя
├── api/                      # Клиенты API (реализация запросов)
│   ├── base_client.py        # Базовый класс для запросов (BaseClient)
│   ├── user_client.py        # Клиент для работы с пользователями (регистрация, авторизация, удаление)
│   ├── orders_client.py      # Клиент для работы с заказами (создание заказа)
│   ├── ingredient_client.py  # Клиент для получения ингредиентов
│   └── endpoint.py           # Эндпоинты, используемые в проекте
└── tests/                    # Тестовые сценарии (pytest)
    ├── conftest.py           # Локальные фикстуры для тестов
    ├── test_api_login.py     # Тесты авторизации (валидные/невалидные данные)
    ├── test_api_order.py     # Тесты заказов (разные сценарии: с ингредиентами/без, с авторизацией/без)
    └── test_api_user.py      # Тесты создания пользователя (уникальный, дубликат, неполные данные)
```
Проект ориентирован на автоматизацию API‑тестирования на Python с использованием pytest.  
Структура позволяет легко масштабировать набор тестов и переиспользовать данные/клиенты между сценариями.  
Тесты покрывают позитивные и негативные сценарии, включая проверку обработки ошибок и граничных условий.  

## **⚙️ Назначение ключевых файлов и папок**  
**1. helpers/, data/ — Утилиты и генераторы данных**  
В этой папке хранятся функции, которые не являются тестами или клиентами API.  
**2. api/ — Клиенты API**  
Здесь реализованы классы для взаимодействия с сервером. Это "обертки" над HTTP-запросами.  
**3. tests/ — Тестовые сценарии**  
Здесь находится логика проверки (Asserts). Тесты используют методы из папки api/.  
conftest.py — Подготовка окружения  
Файл с фикстурами (@pytest.fixture): код, который выполняется до и после тестов.  
**4. Allure-reports/ — Готовые HTML-отчёты**
Сюда помещается итоговый интерактивный отчёт в формате HTML после выполнения команды генерации (allure generate или allure serve). Папка предназначена для хранения артефактов отчётности (например, для CI/CD или ручного просмотра).  

## **🚀 Как запустить тесты**  
**Установите зависимости:**  
pip install -r requirements.txt  
**Запустите все тесты из корневой папки:**  
pytest -v tests/  
**Запустите тесты с генерацией отчета Allure:**  
pytest -v --alluredir=allure-results  
**Сгенерировать HTML-отчет из результатов**  
allure serve allure-results  
**Запустите конкретный файл:**  
pytest -v tests/test_api_user.py
